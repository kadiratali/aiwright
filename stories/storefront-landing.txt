Title: Storefront landing state after login

As a logged-in customer
I want the product catalog to load in a clean initial state
So that I can start shopping without leftover state

Acceptance Criteria:
1. When the "standard" user logs in, the products page is displayed.
2. The page heading reads "Products".
3. On a fresh login, the shopping cart badge is not shown (the cart starts empty).

Technical notes (application under test: https://www.saucedemo.com):
- Login is required first. Reuse the existing steps: "the user is on the login page" and
  "the user logs in as the \"standard\" user". Do NOT redefine them.
- Reuse the existing step "the products page should be displayed" where it fits.
- The existing ProductsPage already exposes: title (.title), inventoryList (.inventory_list),
  and cartBadge (.shopping_cart_badge). Do NOT add new page object methods; reuse these.
- The existing step "the cart badge should not be visible" already asserts the badge is absent.
